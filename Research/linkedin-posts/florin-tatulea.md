## Post 3
Date: ~March 10, 2026 
URL: https://www.linkedin.com/posts/florintatulea_sales-outbound-activity-7437145672820105216-KHfx 
Summary: Lays out a 4-step framework for GTM Engineering applied to outbound: (1) catalog all available signals, (2) map signals to buyer journey stages + lead scoring + ICP cross-reference, (3) create signal-specific messaging and sequences, (4) set up automated routing workflows plus three considerations on TOFU automation, mid-funnel SDR ownership, and multithreading. 
Key Takeaway: Each signal should have its own purpose, messaging, and sequence signal-led outbound fails when you treat all triggers the same instead of mapping them to buyer journey stages.

## Post 4
Date: ~January 13, 2026 
URL: https://substack.com/profile/91059070-florin-tatulea/note/c-199410599 
Summary: Signals and triggers have become a commodity, the signal should NOT be your message; it only tells you it could be the right time. Using a pricing page visit as your opener adds no value beyond what marketing automation already does; reps must paint a contextual picture of why now is the right time to evaluate. 
Key Takeaway: Context is what's missing in outbound, take a signal and paint a picture the exec doesn't already know, rather than just regurgitating the trigger itself. 
Category: Sales Messaging

## Post 5
Date: ~January 2, 2026 
URL: https://www.linkedin.com/posts/florintatulea_your-list-is-your-message-in-outbound-activity-7412928827011551232-eoNS 
Summary: Most orgs build a list then create messaging after it—this should be reversed. Jordan Crawford's PVP (Permissionless Value Prop) framework uses first-party data to build the message first, then the list: an HVAC example pulls permit data on 63,391 installed systems, identifies 4,633 needing replacement worth $23.6M, and surfaces three specific properties with install dates and replacement value. 
Key Takeaway: Your list is your message, build the list with the message in mind by using first-party data to create 1:1 value propositions that make the outreach feel inevitable. 
Category: Cold Email

## Cross-Cutting Themes
Theme 1: Data quality and identity resolution are the invisible foundation-AI, signals, and messaging all fail when the underlying CRM data is fragmented or dirty
Theme 2: Signals are timing indicators, not messages-context and first-party data must be layered on top of commoditized triggers to create differentiated outreach
Theme 3: List construction should be message-first, not message-after—when your data shapes the message, the outreach feels inevitable rather than interruptive
